Table Football Replay System

  • Michael’s documentation about the project

Michael is a designer who likes playing table football and experimenting with Arduinos. He decided to put two light sensors into the football table and connected them with an Arduino so he could detect goals. He asked me if we could hook this up with a webcam and build a replay system

I used a PS-Eye that provided relatively good quality image on 60fps and it had a hacked driver for Windows with a very simple API that allowed me to get images from the camera. I buffered the last 3 seconds in an openFrameworks sketch that was listening to the Arduino on a serial connection and, as it triggered the goal signal, it fed the contents of the buffer to ffmpeg that saved it out as a video file and then replayed it on the screen. If I used all the images on 30fps it showed a half speed playback. After some tweaking, ffmpeg outputted different formats (MOV, FLV) so I could control a Flash movie via a socket that counted the score and played flashy animations over the replay.
